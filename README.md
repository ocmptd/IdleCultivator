# IdleCultivator 修仙挂机

QQ 群修仙挂机游戏机器人。指令极简(`!修炼` → `!突破`),默认 30 分钟自动完成修炼,群活跃度影响修炼效率,深度融合 QQ 群社交场景。

- 技术栈:Java 21 + Maven 单体架构
- 机器人框架:[qqpd-bot-java](https://github.com/Kloping/qqpd-bot-java)
- 持久化:SQLite
- 开发规划:见 [docs/PLAN.md](docs/PLAN.md)

## 快速开始

### 1. 申请机器人凭据

到 <https://q.qq.com/> 申请机器人,获取 `appid` / `token` / `secret`。

### 2. 配置

```bash
cp config.properties.example config.properties
# 编辑 config.properties 填入 appid / token / secret
```

### 3. 构建与运行

```bash
mvn package
java -jar target/idle-cultivator-jar-with-dependencies.jar
```

未配置凭据时,程序会以**本地控制台模式**启动,可直接在终端输入指令调试游戏逻辑:

```
> !创建角色
> !修炼
> !状态
```

## 指令一览(Phase 1)

| 指令 | 说明 |
| --- | --- |
| `!帮助` | 查看指令列表 |
| `!创建角色 [男/女] [称号]` | 创建修仙角色(赠筑基丹×1) |
| `!改称号 [新称号]` | 修改称号,每 2 天可改一次 |
| `!修炼 [功法] [时长]` | 开始修炼(默认 18~28 分钟随机,最长 2 小时);时长越长修为/灵石越多且每小时效率越高(约 30m→62 / 1h→150 / 2h→400 修为,灵石=修为÷2);≤30 分钟自动收获,更长时长需手动收获;修炼中重复输入提示剩余分钟 |
| `!收获` | 收获修炼;满 10 分钟可提前结束(收益按进度衰减,越接近完成扣得越少);超时溢出按分级惩罚转化(≤1h 灵尘 / 1-3h 残破法宝 / >3h 灵石+20% 走火入魔) |
| `!功法` | 查看内置功法及收益倍率(吐纳诀/紫府诀/玄天功/太虚剑意,按境界解锁) |
| `!背包` | 查看背包物品、灵石与装备 |
| `!突破 [用丹]` | 阶段末(20/40/60 级)且修为足够时突破进入下一阶段(基础 80%,带「用丹」消耗筑基丹提升至 95%) |
| `!状态` | 查看等级、属性(气血/攻击/防御/神识)、背包、姿容与修炼进度 |
| `!形象描述 [道具]` | 查看角色(或背包道具)的图片描述词(AI 绘图 prompt,命中缓存则不重复生成,未命中时异步请求已配置的生图 API) |

指令前缀可在 `config.properties` 中通过 `command.prefix` 配置(可为空);修炼到期/结算结果会主动推送到所在群,无法推送时(QQ 被动消息仅 5 分钟时效)自动结算加经验(不会因此受溢出惩罚),结算信息暂存,玩家下次发任意消息时机器人先补发【修炼结算】再回复。

## 等级与属性

- 80 级 4 个阶段(每 20 级一阶段,与境界合并):练气 1~20 / 筑基 21~40 / 金丹 41~60 / 元婴 61~80
- 修为攒够自动升级(可连升,溢出结转);阶段末(20/40/60 级)需 `!突破` 才能继续
- 节奏(按日均约 2000 修为 ≈ 每天约 5 次 2h 修炼):练气 ≈2 天,筑基 ≈20 天,金丹+元婴 ≈40 天(元婴期修炼收益×0.8 境界压制)
- 属性随等级/阶段成长:气血 100+20/级;攻击 10+3/级(每阶段额外+20);防御 5+2/级(每阶段额外+15);神识 10+2/级;性别加成:男 气血/防御+5%,女 神识+5%
- 赛季制后续版本开放,已预留 season 字段(详见 docs/LEVEL-DESIGN.md)
